> **目标读者**：负责具体编码的 Agent / 开发者。本文档自包含，按本文档即可完成编码，无需其他上下文。
> **参考样板**：本项目 **"微信公众号"模块**（账号池 / 公众号管理 / 文章下载 / 网址下载 / 下载历史 五个页面）。
> **数据采集原理**：移植自开源项目 [XHS-Downloader](https://github.com/JoeanAmier/XHS-Downloader)，其源码关键逻辑已在附录 A 完整给出，**实现时无需再去读该仓库**。

## 1. 总体目标

在现有工具（Flask 后端 + 原生 JS Hash 路由 SPA）中新增 **"小红书"侧边栏菜单分组**，对标公众号模块的页面与功能。原则：**能实现的都要有，受签名/登录限制无法实现的明确不做**。

### 1.1 页面映射关系（公众号 → 小红书）

| 公众号模块页面 | 小红书对应页面 | 路由 key | 可行性 | 差异说明 |
|---|---|---|---|---|
| 账号池（扫码登录，`login.js`） | Cookie 登录 | `xhs_login` | ✅ | 小红书 Cookie 非强制（无 Cookie 视频仅低清晰度），单 Cookie 即可，无需做成多账号池。支持「弹窗扫码自动提取」+「手动粘贴 Cookie」两种方式 |
| 公众号管理（搜索+收藏，`accounts.js`） | 博主管理 | `xhs_accounts` | ✅（搜索除外） | 小红书搜索 API 需 x-s 签名，**不做搜索**；改为「粘贴博主主页链接 → 解析博主信息 → 收藏」 |
| 文章下载（选号→文章列表→勾选下载，`articles.js`） | 笔记下载 | `xhs_notes` | ⚠️ 仅首屏 | 选择已收藏博主 → 列出其主页**首屏约 30 条笔记**（解析主页 HTML，无需签名）→ 勾选批量下载。全量分页需 x-s 签名，列为可选进阶（见 §7） |
| 网址下载（粘贴 URL 批量下载，`download.js`） | 链接下载 | `xhs_download` | ✅ | 粘贴笔记链接（支持多行、短链、分享口令文本），解析后下载无水印图片/视频/Live 图。这是**本模块核心页面** |
| 下载历史（`history.js`） | 下载历史 | `xhs_history` | ✅ | 与公众号下载历史同构：表格 + 打开文件/目录 + 删除单条（连文件）/清空 |
| RSS 订阅（公众号附属功能） | — | — | ❌ 不做 | 依赖全量分页采集，首版不做 |

### 1.2 明确不做的功能（受 x-s 签名 / 风控限制）

- 站内搜索（笔记搜索、用户搜索）
- 博主全量作品分页采集（首版仅首屏，进阶方案见 §7）
- 账号收藏列表 / 点赞列表采集
- 评论采集
- 剪贴板监听（XHS-Downloader 有此功能，桌面 Web 工具场景下价值低，不做）

- 组件是全局对象常量（如 `XhsLoginPage`），`render()` 返回 HTML 字符串，事件用 `onclick="XhsLoginPage.xxx()"` 内联绑定；

### 3.1 新增文件

### 3.2 数据存储

| 项 | 路径 / Key | 说明 |
|---|---|---|
| 下载根目录 | `DATA_DIR / "xhs_downloads"` | 结构：`xhs_downloads/<博主昵称>/<发布日期_标题>/`，图集为 `01.png、02.png…` + Live 图 `01.mp4…`；视频笔记为 `<标题>.mp4` |
| 收藏博主 | `DATA_DIR / "xhs_accounts.json"` | list，每项见 §3.5 |
| 下载历史 | `DATA_DIR / "xhs_history.json"` | list，每项 `{title, note_id, type(视频/图文), author, path, size, time, success, error}` |
| Cookie | `app_settings.json` 中 `xhs_cookie` | 字符串，经 `get_settings()/save_settings()` 读写 |
| 图片格式 | `app_settings.json` 中 `xhs_image_format` | 默认 `"png"`，可选 `png/jpeg/webp/auto`（见附录 A.4） |
| Live 图开关 | `app_settings.json` 中 `xhs_download_live` | 默认 `true` |

### 3.3 `XhsClient` 类（backend/xiaohongshu.py 内）

无状态请求客户端，`requests.Session`，所有公开方法线程安全（不共享可变状态即可）。

```
常量:
  USER_AGENT = Chrome 桌面版 UA（与 douyin.py 的 USER_AGENT 一致即可）
  BASE_HEADERS = {
      "accept": "text/html,application/xhtml+xml,...",       # 普通浏览器 accept
      "referer": "https://www.xiaohongshu.com/explore",
      "user-agent": USER_AGENT,
  }
  # 每次请求时若 settings 中有 xhs_cookie，则追加 "cookie" 头

方法:
  extract_links(text) -> list[str]        # 从任意文本提取作品链接（含短链还原），见附录 A.2
  resolve_short_link(url) -> str          # 请求 xhslink.com 短链，follow_redirects 取最终 URL
  get_initial_state(url) -> dict          # 请求页面 HTML → 提取 window.__INITIAL_STATE__ → dict，见附录 A.3
  get_note_detail(url) -> dict            # get_initial_state + 提取笔记数据 + 构造无水印下载地址，返回结构见 §3.4
  get_user_profile(url) -> dict           # 解析博主主页：返回博主信息 + 首屏笔记列表，见 §3.5
  download_file(url, save_path) -> int    # requests 流式下载（参考 douyin.py 的 download_file），返回字节数
```

**防风控要求**：
- 每次 HTTP 请求之间 `time.sleep(random.uniform(1.0, 2.5))`；
- 失败重试最多 3 次（复用 `get_settings()["max_retries"]`）；
- 走代理：`get_proxies_dict()`，请求后 `report_proxy_status(proxy_url, success)`（参考 `accounts.py` 的用法）；
- 收到 461/406/空 `noteDetailMap` 时，向上抛出明确错误信息：「触发风控或 Cookie 失效，请在登录页更新 Cookie」。

### 3.4 笔记数据结构（`get_note_detail` 返回值，也是 /parse 接口响应）

```json
{
  "note_id": "65f1a2b3...",
  "note_url": "https://www.xiaohongshu.com/explore/65f1a2b3...?xsec_token=...",
  "title": "标题",
  "desc": "正文描述",
  "type": "视频 | 图文 | 图集",
  "tags": ["标签1", "标签2"],
  "author": {"user_id": "...", "nickname": "...", "avatar": "...", "url": "https://www.xiaohongshu.com/user/profile/<user_id>"},
  "stats": {"liked": "1.2万", "collected": "3456", "comment": "789", "share": "100"},
  "publish_time": "2024-03-13 12:00:00",
  "timestamp": 1710302400,
  "images": ["https://ci.xiaohongshu.com/<token>?imageView2/format/png", "..."],
  "lives": ["https://sns-video-....mp4 或 null，与 images 等长", null],
  "video": "https://sns-video-bd.xhscdn.com/<originVideoKey> 或 null",
  "cover": "images[0] 或视频封面"
}
```

字段提取路径、无水印地址构造规则见附录 A.4 / A.5（**必须严格按附录实现**）。

### 3.5 博主数据结构（`get_user_profile` 返回值）

请求博主主页 `https://www.xiaohongshu.com/user/profile/<user_id>?xsec_token=...`，同样走 `get_initial_state`，数据在 `state["user"]`：

```json
{
  "user": {
    "user_id": "5ff0e6410000000001008400",
    "nickname": "昵称",            // user.userPageData.basicInfo.nickname
    "avatar": "...",               // user.userPageData.basicInfo.images
    "desc": "简介",                // user.userPageData.basicInfo.desc
    "red_id": "小红书号",          // user.userPageData.basicInfo.redId
    "fans": "1.2万",               // user.userPageData.interactions 中 type=="fans" 的 count
    "url": "原始主页链接（含 xsec_token，原样保存，刷新笔记时复用）"
  },
  "notes": [
    {
      "note_id": "...",            // item.id
      "xsec_token": "...",         // item.xsecToken —— 必须保存，下载时拼链接用
      "title": "...",              // item.noteCard.displayTitle
      "cover": "...",              // item.noteCard.cover.urlDefault
      "type": "video | normal",    // item.noteCard.type
      "liked": "..."               // item.noteCard.interactInfo.likedCount
    }
  ]
}
```

**注意**：`state["user"]["notes"]` 实际结构是「数组的数组」（按 tab 分组），需展开第一层后再逐项取 `noteCard`；逐字段 `.get()` 容错，结构变动时返回已解析出的部分并附 `warning` 字段。

收藏博主时把 `user` 对象（含 `added_time`）追加进 `xhs_accounts.json`，以 `user_id` 去重 —— 与 `backend/accounts.py` 的 add/remove/list 完全同构。

### 3.6 API 端点设计

#### 蓝图 `xhs_auth_bp`，前缀 `/api/xhs-auth`（backend/xiaohongshu_login.py）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/status` | 返回 `{cookie_set: bool, logged_in: bool, login_state: {...}}`。`logged_in` 判定：cookie 中含 `web_session` 字段。**不做网络校验**（小红书校验接口需签名），仅本地判断 |
| POST | `/login` | 启动 Playwright 扫码登录线程（非 headless，`launch_chromium`），打开 `https://www.xiaohongshu.com`，轮询 `context.cookies()` 直到出现 `web_session`（240 秒超时），拼 `name=value; ...` 存入 settings `xhs_cookie`。状态机 `_login_state {status: idle/scanning/success/failed, message, progress}` 照抄 `douyin_login.py` |
| POST | `/save-cookie` | body `{cookie}`，手动粘贴保存；校验非空且包含 `web_session`（不含时仍保存但返回 warning 提示「游客 Cookie，视频可能低清晰度」） |
| POST | `/logout` | 清空 settings 中 `xhs_cookie` |

#### 蓝图 `xhs_bp`，前缀 `/api/xhs`（backend/xiaohongshu.py）

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/parse` | body `{url}`，单链接解析，返回 §3.4 结构（**不下载**，前端预览用） |
| POST | `/download` | body `{urls: "多行文本或数组"}`，先 `extract_links` 提取全部链接，创建任务下载，返回 `{task_id, message, count}` |
| GET | `/accounts` | 收藏博主列表 `{accounts: [...], total}` |
| POST | `/accounts/parse` | body `{url}`，解析博主主页返回 §3.5 结构（预览，不收藏） |
| POST | `/accounts` | body 为 §3.5 的 `user` 对象，收藏（user_id 去重） |
| DELETE | `/accounts/<user_id>` | 取消收藏 |
| GET | `/accounts/<user_id>/notes` | 用收藏时保存的 `url` 重新请求主页，返回最新首屏笔记列表（§3.5 的 `notes`） |
| POST | `/download-notes` | body `{notes: [{note_id, xsec_token, title}], account_name}`，批量下载选中笔记，返回 `{task_id}`。每条拼 `https://www.xiaohongshu.com/explore/<note_id>?xsec_token=<token>&xsec_source=pc_user` 后走与 `/download` 相同的下载管线 |
| GET | `/download-status/<task_id>` | 任务状态轮询 |
| POST | `/download-cancel/<task_id>` | 请求取消任务 |
| GET | `/history` | `{history: [...], total}`，倒序，`?limit=50` |
| DELETE | `/history` | 清空历史（不删文件） |
| DELETE | `/history/<int:index>` | 删除单条 + 删除对应文件/文件夹（照抄 `articles.py::delete_history_item`） |
| POST | `/open-folder` `/open-file` `/open-parent` | 照抄 `articles.py` 同名实现，根目录换成 `xhs_downloads` |

#### 下载任务状态机（照抄 articles.py 模式）

```python
_download_tasks = {}          # task_id -> task dict
_download_lock = threading.Lock()

task = {
  "status": "running | cancelling | cancelled | completed | failed",
  "total": N, "completed": 0, "failed": 0, "skipped": 0,
  "current": "正在下载：xxx",
  "results": [{"title":..., "success":..., "path":..., "error":...}],
  "start_time": ..., "end_time": ..., "cancel_requested": False,
}
```

- `task_id = f"xhs_{int(time.time())}"`；线程 `daemon=True`；
- 每个笔记下载前检查 `cancel_requested`；
- 已下载去重：历史中存在相同 `note_id` 且 `success=True` 时跳过并计入 `skipped`（对应 XHS-Downloader 的「自动跳过已下载」）；
- 单条笔记下载流程：`get_note_detail` → 建目录 `xhs_downloads/<clean(作者昵称)>/<日期_clean(标题)>/` → 视频下载 `.mp4` / 图集逐张下载 `01.png…` + Live 图 `01.mp4…` → 写入历史（无论成败）。
- 文件名清洗复用 `douyin.py::clean_filename` 的逻辑（拷贝一份私有函数，**不要跨模块 import 抖音代码**）。

### 4.2 各页面规格

#### ① Cookie 登录（xhs_login.js，参考 login.js + douyin/dy_login.js）

- 状态卡片：当前 Cookie 状态（未配置 / 已配置游客 Cookie / 已登录），由 `API.xhs.auth.status()` 渲染；
- 「扫码登录」按钮 → `POST /api/xhs-auth/login` → 每 2 秒轮询 `/status` 中 `login_state`，显示进度文案；success 后刷新状态卡片；
- 「手动粘贴 Cookie」折叠区：textarea + 保存按钮 → `/save-cookie`；附获取 Cookie 教程文案（浏览器 F12 → Network → 复制 cookie 请求头）；
- 「退出登录」按钮（双重 `Modal.confirm`）；
- 提示文案：Cookie 非必需，但未配置时视频只能下载低清晰度。

#### ② 博主管理（xhs_accounts.js，参考 accounts.js）

- 顶部「添加博主」输入框：粘贴博主主页链接 → `API.xhs.parseUser(url)` → 弹出预览卡片（头像/昵称/小红书号/粉丝/简介）→ 确认收藏 `API.xhs.addAccount(user)`；
- 收藏列表卡片网格（同 accounts.js 的卡片风格）：头像、昵称、小红书号、粉丝数；
- 卡片操作：「查看笔记」→ `Router.navigate('xhs_notes', {user_id})`；「删除」→ `Modal.confirm` → `API.xhs.removeAccount(user_id)`；
- 头像图片可能因防盗链裂图：`<img>` 加 `onerror` 兜底占位（index.html 已有 `referrer no-referrer`，一般可直接显示）。

#### ③ 笔记下载（xhs_notes.js，参考 articles.js）

- 顶部博主下拉选择器（数据来自收藏列表；支持 `init(params)` 接收 `?user_id=` 直接选中）；
- 选中后 `API.xhs.listNotes(user_id)` 加载首屏笔记，**封面图网格**展示（非表格，笔记是视觉内容）：封面、标题、类型徽标（视频/图文）、点赞数、勾选框；
- 工具栏：全选 / 取消全选 / 「下载选中 (N)」按钮；
- 显著提示条：「当前仅能获取博主主页首屏约 30 条笔记」；「刷新」按钮重新拉取；
- 下载选中 → `API.xhs.downloadNotes(notes, accountName)` → 得到 `task_id` → 复用 §4.3 的进度面板组件逻辑。

#### ④ 链接下载（xhs_download.js，参考 download.js —— 本模块核心页面）

- 多行 textarea：「每行一个链接；支持 explore / discovery 链接、xhslink 短链、整段分享口令文本」；
- 「解析预览」按钮（可选操作）：取第一个链接 `API.xhs.parse(url)`，展示预览卡片（封面、标题、作者、类型、点赞收藏数、图集张数）；
- 「开始下载」按钮：`API.xhs.download(urls)` → `task_id` → 进度面板（§4.3）；
- 下载完成后 Toast 汇总：成功 N / 失败 N / 跳过 N。

#### ⑤ 下载历史（xhs_history.js，参考 history.js + 公众号历史的删除交互）

- 统计卡片：总数 / 视频数 / 图文数（参考 dy_downloads 统计行，但风格用浅色主题）；
- 表格列：标题、类型徽标、作者、大小、时间、操作（打开文件 / 打开目录 / 删除）；
- 删除单条：`Modal.confirm` 提示「将同时删除本地文件」→ `API.xhs.deleteHistory(index)`；
- 清空历史：双重确认，仅清记录不删文件；
- 「打开下载文件夹」按钮 → `/open-folder`；
- `onShow()` 钩子里刷新列表（路由缓存命中时自动更新）。

### 4.3 共用下载进度面板（约定，非独立文件）

`xhs_download.js` 与 `xhs_notes.js` 各自内联实现相同逻辑（与 download.js 的 `_pollTimer` 模式一致）：

- 每 1 秒 `API.xhs.downloadStatus(taskId)`；
- 进度条 `completed+failed+skipped / total`，当前项文案 `current`；
- 「取消」按钮 → `API.xhs.cancelDownload(taskId)`；
- `status` 进入 `completed/failed/cancelled` 时停止轮询并 Toast；

## 6. 风险与注意事项

- **`__INITIAL_STATE__` 结构变动风险**：所有字段提取必须逐层 `.get()` 容错；解析失败时返回 `{"error": "解析失败，可能页面结构已更新或触发风控"}`，不许抛裸异常 500。
- **xsec_token 必带**：2024 年后小红书链接不带 `xsec_token` 参数会 404/跳转。`extract_links` 不得截断 query 参数；从主页列表拼笔记链接必须用保存的 `xsecToken`。
- **请求频率**：批量下载时相邻笔记间 sleep 1.5~3 秒随机；图片文件下载间 0.3~0.8 秒。
- **Cookie 失效表现**：返回页面无 `noteDetailMap` 数据或重定向到登录页 —— 错误信息要引导用户去登录页更新 Cookie。
- **YAML 解析**：`__INITIAL_STATE__` 是 JS 对象字面量，含 `undefined`，`json.loads` 会失败，必须按附录 A.3 处理。
- **代理**：所有对 xiaohongshu.com / xhscdn.com 的请求都走 `get_proxies_dict()`（可能为 None，正常直连）。

## 7. 进阶（首版不做，仅记录）

- **博主全量笔记分页**：API `GET https://edith.xiaohongshu.com/api/sns/web/v1/user_posted?num=30&cursor=...&user_id=...` 需要 `x-s/x-t` 签名头。XHS-Downloader 用 PyPI 包 `xhshow` 的 `Xhshow().sign_headers_get(uri, cookies, params)` 生成（见其 `source/application/user_posted.py`）。若后续要做：`pip install xhshow`，在 `XhsClient` 加 `get_user_posted_page(user_id, cursor)`，`/accounts/<id>/notes` 加 `?cursor=` 分页参数即可，前端加「加载更多」按钮。
- 搜索、收藏/点赞列表采集：同样依赖签名，做完分页后顺势可做。

# 附录 A：XHS-Downloader 核心采集逻辑（已验证，照此实现）

> 提炼自 XHS-Downloader 源码（source/application/、source/expansion/converter.py），是本模块数据采集的**唯一权威依据**。

## A.1 请求头

```python
HEADERS = {
    "accept": "text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.7",
    "referer": "https://www.xiaohongshu.com/explore",
    "user-agent": "<Chrome 桌面版 UA>",
    # 有 cookie 时: "cookie": "<settings.xhs_cookie>"
}
```

## A.2 链接识别正则

```python
LINK  = r"(?:https?://)?www\.xiaohongshu\.com/explore/\S+"          # 普通作品链接
SHARE = r"(?:https?://)?www\.xiaohongshu\.com/discovery/item/\S+"   # 分享作品链接
USER_NOTE = r"(?:https?://)?www\.xiaohongshu\.com/user/profile/[a-z0-9]+/\S+"  # 主页内笔记链接
SHORT = r"(?:https?://)?xhslink\.com/[^\s\"<>\\^`{|}，。；！？、【】《》]+"      # 短链（注意中文标点截断）
```

`extract_links(text)` 流程：按空白分词 → 每段先匹配 SHORT（命中则 GET 请求 follow_redirects=True 还原成长链）→ 再依次匹配 SHARE / LINK / USER_NOTE，命中即收集。note_id 提取：`re.search(r"(?:explore|item)/([0-9a-z]+)", url)`。

## A.3 HTML → 数据对象（converter 逻辑）

1. GET 作品/主页 URL 得到 HTML；
2. `lxml.etree.HTML(html).xpath("//script/text()")` 取全部 script 文本，**倒序**找第一个以 `window.__INITIAL_STATE__` 开头的；
3. 去掉前缀 `window.__INITIAL_STATE__=`；
4. 清掉非法控制字符：`re.sub(r"[\x00-\x08\x0b\x0c\x0e-\x1f\x7f]", "", text)`；
5. **用 `yaml.safe_load(cleaned)` 解析**（不是 json.loads！因为文本含 `undefined`，YAML 会把它解析为字符串而不报错。本项目 venv 已有 PyYAML）；
6. 笔记页数据：`state["note"]["noteDetailMap"]` 是 dict，**取最后一个 value** 的 `["note"]`（即 `list(noteDetailMap.values())[-1]["note"]`）；移动端分享页变体路径为 `state["noteData"]["data"]["noteData"]`，先试它、再试 PC 路径；
7. 主页数据：`state["user"]`。

## A.4 笔记字段提取（note 对象内）

| 目标字段 | 路径 |
|---|---|
| note_id | `noteId` |
| 标题 / 描述 | `title` / `desc` |
| 标签 | `tagList[*].name` 拼接 |
| 点赞/收藏/评论/分享 | `interactInfo.likedCount / collectedCount / commentCount / shareCount` |
| 发布时间 | `time`（毫秒时间戳）→ `datetime.fromtimestamp(t/1000)` |
| 作者 | `user.nickname`（备选 `user.nickName`）、`user.userId` |
| 类型判定 | `type=="video"` 且 `len(imageList)==1` → 视频；`type=="video"` 且多图 → 图集；`type=="normal"` → 图文 |

**图片无水印地址**（对 `imageList` 每项）：
1. 取 `urlDefault`（为空则取 `url`）；
2. token = `"/".join(url.split("/")[5:]).split("!")[0]`（URL 第 6 段起为 token，去掉 `!` 后缀的水印样式参数）；
3. 指定格式：`https://ci.xiaohongshu.com/{token}?imageView2/format/{png|jpeg|webp}`；auto 格式：`https://sns-img-bd.xhscdn.com/{token}`；
4. 最后 `bytes(url,"utf-8").decode("unicode_escape")` 处理转义。

**Live 图地址**（与 imageList 等长，无则 None）：`imageList[i].stream.h264[0].masterUrl`。

**视频无水印地址**：
1. 首选：token = `note.video.consumer.originVideoKey` → `https://sns-video-bd.xhscdn.com/{token}`；
2. 兜底：合并 `note.video.media.stream.h264 + h265` 列表，按 `height` 取最大项的 `backupUrls[0]` 或 `masterUrl`。

## A.5 主页首屏笔记

`state["user"]["notes"]` 为按 tab 分组的嵌套数组，展开后每项：`id`、`xsecToken`、`noteCard.displayTitle`、`noteCard.type`、`noteCard.cover.urlDefault`、`noteCard.interactInfo.likedCount`。拼回笔记链接：`https://www.xiaohongshu.com/explore/{id}?xsec_token={xsecToken}&xsec_source=pc_user`。

## A.6 文件下载

直接 GET（带 HEADERS，不带 cookie 也可），流式写盘；图片扩展名按所选 format，视频/Live 图为 `.mp4`。先写 `.tmp` 再 rename，避免半截文件入库。
